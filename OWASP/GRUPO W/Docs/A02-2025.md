**A02:2025- SECURITY MISSCONFIGURATION**


![SECURITY-MISCONFIGURATION-ATTACK-EXAMPLE](https://github.com/user-attachments/assets/201ea52c-aeeb-4e66-b366-3d178e91fb6f)


**Descripción:** La Configuración de Seguridad Incorrecta es una de las vulnerabilidades más comunes y se refiere a la falta de una configuración de seguridad adecuada en cualquier parte de la pila de una aplicación, incluyendo el sistema operativo, el servidor web, el servidor de aplicaciones, la base de datos, el framework, las bibliotecas y el propio código. Esta vulnerabilidad surge de configuraciones predeterminadas inseguras, configuraciones incompletas o erróneas, o la falta de mantenimiento de la configuración de seguridad.

**Causas:**

- Configuraciones predeterminadas de fábrica inseguras que no se cambian.

-  Error humano al configurar los servicios.

- Falta de parches de seguridad o actualizaciones.

- Exposición de información sensible en mensajes de error detallados.

- Servicios innecesarios habilitados.

- permisos de archivos y directorios incorrectos.

**Impacto Potencial:**
 Puede llevar a la exposición de datos sensibles, acceso no autorizado a sistemas, escalada de privilegios, o incluso el control completo del servidor subyacente. Los atacantes pueden aprovechar estas configuraciones incorrectas para lanzar otros ataques.

**Métodos de Explotación:** Los atacantes pueden buscar:

- **Credenciales por Defecto:** Intentar acceder a sistemas usando nombres de usuario y contraseñas predeterminados para bases de datos, aplicaciones o servicios.

- **Exposición de Mensajes de Error Detallados:** Provocar errores para obtener información valiosa sobre la estructura interna de la aplicación o el sistema.

- **Puertos y Servicios Innecesarios Expuestos:** Escanear puertos para encontrar servicios que no deberían estar accesibles públicamente y explotar vulnerabilidades conocidas en ellos.

- **Permisos de Archivos Incorrectos:** Acceder a archivos que deberían estar protegidos (ej., archivos de configuración, logs, copias de seguridad).

- **Directorio de Aplicación Abierto:** Acceder a listados de directorios o archivos ocultos que pueden contener información sensible o código fuente.

**Ejemplo de Ataque Real:**
 Un atacante podría encontrar un servidor web configurado con un directorio de administración accesible públicamente y con credenciales predeterminadas, permitiéndole obtener control total del sitio. Otro ejemplo es la exposición de archivos de copia de seguridad o configuración con datos sensibles.

**Herramientas y Técnicas:**

**Nmap:** Para escanear puertos y servicios.

**Nikto/DirBuster:** Para encontrar directorios y archivos ocultos.

**Burp Suite/OWASP ZAP:** Para analizar respuestas HTTP y encontrar mensajes de error detallados.

**Shodan/Censys:** Para buscar sistemas con configuraciones inseguras expuestos en internet.

**Capital One (2019) - Error en el WAF (Firewall)**

-   **El incidente:** Un atacante accedió a los datos de **100 millones de clientes**.
-   **La falla:** Un Firewall de Aplicaciones Web (WAF) estaba mal configurado, lo que permitió un ataque de tipo 
- **SSRF** (_Server-Side Request Forgery_). El firewall tenía permisos excesivos que le permitían consultar el servicio de metadatos de AWS y obtener credenciales de administrador.
-   **Impacto:** Robo de números de seguridad social y cuentas bancarias. Se debió a una configuración de permisos demasia
